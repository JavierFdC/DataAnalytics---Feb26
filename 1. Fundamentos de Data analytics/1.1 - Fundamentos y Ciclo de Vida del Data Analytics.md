## Objetivos de la sesión

## 1. ¿Qué es el Data Analytics?

El Data Analytics es el proceso de examinar datos para extraer conclusiones que apoyen decisiones. No es hacer gráficos bonitos ni llenar dashboards de números — es **reducir la incertidumbre antes de decidir**.

La diferencia entre una organización que usa datos y una que no los usa no está en la cantidad de datos que tiene. Está en la capacidad de convertir esos datos en preguntas respondidas.

**Qué es:**
- Transformar datos brutos en información accionable
- Responder preguntas de negocio con evidencia cuantitativa
- Apoyar decisiones que antes se tomaban por intuición o experiencia

**Qué no es:**
- Recopilar datos por acumular
- Hacer un dashboard que nadie consulta
- Automatizar un informe que ya era inútil en papel

## 2. El ciclo de vida del dato

El dato recorre un camino desde que se genera hasta que alguien toma una decisión basándose en él. Ese camino tiene fases bien definidas.

| Fase | Qué ocurre | Quién interviene |
|------|-----------|-----------------|
| **1. Generación** | El dato se crea: un sensor, un formulario, una transacción, un log | Sistema o proceso de negocio |
| **2. Ingesta** | El dato se captura y se mueve al entorno analítico | Ingeniería de datos |
| **3. Almacenamiento** | El dato se guarda en un formato estructurado y accesible | Ingeniería de datos / DBA |
| **4. Transformación** | El dato se limpia, se estructura y se enriquece | Ingeniería de datos / Analista |
| **5. Análisis** | Se extraen patrones, tendencias y conclusiones | Analista de datos |
| **6. Visualización** | Los resultados se presentan de forma comprensible | Analista de datos / BI |
| **7. Decisión** | Un responsable actúa en base a lo analizado | Negocio |

> El ciclo no es lineal. En la práctica se repite, se itera y con frecuencia la fase 7 genera nuevas preguntas que reinician el proceso desde la fase 5.

### El ciclo en MetroBus

MetroBus genera datos en cada expedición: hora de salida, ocupación, consumo de combustible, incidencias. Esos datos pasan por todas las fases antes de llegar a la pantalla del director de operaciones:

1. Los sensores y conductores registran los datos en cada viaje *(generación)*
2. Los ficheros CSV se consolidan cada noche *(ingesta)*
3. Se almacenan en la base de datos analítica *(almacenamiento)*
4. Se limpian errores y se calculan métricas derivadas *(transformación)*
5. Se detecta que las líneas nocturnas tienen un 34% más de retraso *(análisis)*
6. Se construye un dashboard de puntualidad por línea y franja horaria *(visualización)*
7. La dirección decide reforzar la frecuencia en las líneas más retrasadas *(decisión)*

## 3. Roles en un equipo de datos

En proyectos reales, el trabajo analítico se reparte entre varios perfiles. Conocerlos ayuda a entender dónde encaja el trabajo de un analista.

| Rol | Responsabilidad principal | Herramientas habituales |
|-----|--------------------------|------------------------|
| **Data Engineer** | Construir y mantener los pipelines que mueven y almacenan los datos | Python, SQL, Spark, Airflow |
| **Data Analyst** | Transformar datos en respuestas a preguntas de negocio | SQL, Excel, Power BI, Python |
| **Data Scientist** | Construir modelos estadísticos y de machine learning | Python, R, Jupyter |
| **BI Developer** | Diseñar y mantener cuadros de mando e informes | Power BI, Tableau, Qlik |
| **Data Steward** | Garantizar la calidad, seguridad y gobernanza del dato | Herramientas de catalogación y MDM |

> En equipos pequeños —y en este curso— el analista asume tareas de varios de estos roles. Es habitual.

## 4. El ecosistema de herramientas

Las herramientas no son el fin; son el medio. Lo que determina si un análisis es bueno no es la herramienta que se usó, sino la pregunta que respondió.

```
FUENTE DE DATOS
     │
     ▼
[Ingesta y almacenamiento]   → SQL, bases de datos, data warehouses
     │
     ▼
[Transformación y limpieza]  → Excel / Power Query, Python / Pandas
     │
     ▼
[Análisis]                   → Excel, Python, SQL
     │
     ▼
[Visualización]              → Excel, Power BI, Tableau, Qlik Sense
     │
     ▼
DECISIÓN DE NEGOCIO
```

Las herramientas que cubriremos en el curso tienen posición en este esquema:

| Herramienta | Fase principal | Cuándo elegirla |
|-------------|---------------|-----------------|
| **Excel** | Transformación, análisis, visualización | Datasets pequeños, trabajo individual, prototipado rápido |
| **Power BI** | Visualización y cuadros de mando | Informes recurrentes, datos que se actualizan, audiencias grandes |
| **Python / Pandas** | Transformación y análisis | Datasets grandes, automatización, análisis estadístico avanzado |
| **SQL** | Almacenamiento y consulta | Cuando los datos viven en una base de datos relacional |

## 5. Por qué fallan los proyectos de datos

La mayoría de los proyectos de datos no fracasan por razones técnicas. Fracasan por razones de negocio.

| Causa de fracaso | Descripción |
|-----------------|-------------|
| **Pregunta mal definida** | El análisis responde algo, pero no lo que el negocio necesita saber |
| **Dato de mala calidad** | Los resultados son incorrectos pero nadie lo detecta hasta que el daño está hecho |
| **Resultado no comunicado** | El análisis es correcto pero nadie en negocio lo entiende o lo usa |
| **Sin propietario en negocio** | Nadie es responsable de validar los KPIs ni de actuar sobre los resultados |
| **Herramienta equivocada** | Se usa una herramienta compleja para un problema simple, o al revés |
