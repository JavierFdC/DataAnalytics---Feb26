# Proyecto Práctico: Dashboard Completo en Excel

## Descripción del proyecto

En este proyecto aplicarás todo lo aprendido en el Bloque 4 para construir un dashboard de ventas completo y profesional en Excel, integrando Microsoft 365 Copilot como herramienta de apoyo en cada fase del proceso.

**Dataset:** `datasets/ventas_retail.xlsx`  
**Entregable:** Archivo Excel con el dashboard terminado  
**Formato del libro:** mínimo 3 hojas (Datos, Cálculos/Tablas, Dashboard)

---

## Briefing del proyecto

Eres analista de datos en una empresa de retail. El equipo de dirección necesita un **dashboard ejecutivo de ventas 2024** que les permita responder las siguientes preguntas clave:

1. ¿Cuál es el rendimiento global de ventas del año?
2. ¿Qué categorías y productos generan más ingresos?
3. ¿Cómo evolucionan las ventas a lo largo del año?
4. ¿Qué vendedor y qué región tienen mejor desempeño?
5. ¿Hay diferencias significativas entre el canal Online y la Tienda física?

El dashboard debe ser **interactivo**: el equipo directivo debe poder filtrar por canal, región y período sin necesidad de saber Excel.

---

## Fases del proyecto

### Fase 1 — Preparación y limpieza de datos (sesión 4.1)

**Objetivo:** tener los datos listos para analizar.

Tareas:
- [ ] Convertir el rango en tabla de Excel (Ctrl+T)
- [ ] Identificar y corregir errores de limpieza:
  - Espacios extra en nombres de vendedores
  - Fechas en formato incorrecto
  - Celdas vacías en Total_Venta
  - Filas duplicadas
- [ ] Verificar que todos los tipos de dato son correctos (fechas como fecha, números como número)
- [ ] Documentar qué problemas encontraste y cómo los corregiste

**Copilot en esta fase:**
```
"¿Hay inconsistencias en la columna Vendedor? Muéstrame qué valores distintos existen"
```
```
"Resalta en rojo las filas donde Total_Venta esté vacío"
```

---

### Fase 2 — Análisis con tablas dinámicas (sesión 4.2)

**Objetivo:** responder las preguntas de negocio con tablas dinámicas.

Crea las siguientes tablas dinámicas en una hoja llamada `Cálculos`:

| Tabla dinámica | Filas | Columnas | Valores |
|---------------|-------|----------|---------|
| TD1 | Categoría | — | Total_Venta (suma), Unidades (suma) |
| TD2 | Fecha (por Mes) | Canal | Total_Venta (suma) |
| TD3 | Vendedor | Región | Total_Venta (suma) |
| TD4 | Categoría | Canal | Total_Venta (suma), % del total |
| TD5 | Fecha (por Trimestre) | — | Total_Venta (suma) |

Añade un campo calculado `Ticket_Medio = Total_Venta / Unidades` en la TD3.

**Copilot en esta fase:**
```
"Crea una tabla dinámica con las ventas totales por Categoría y Canal"
```
```
"¿Qué trimestre concentra más ventas? Muéstralo con una tabla dinámica por trimestre"
```

---

### Fase 3 — Visualizaciones (sesión 4.3)

**Objetivo:** transformar las tablas dinámicas en gráficos claros para el dashboard.

Crea los siguientes gráficos (guárdalos en la hoja `Cálculos` o directamente en el Dashboard):

| Gráfico | Tipo | Origen |
|---------|------|--------|
| Ventas por categoría | Barras horizontales ordenado mayor → menor | TD1 |
| Evolución mensual de ventas | Líneas + marcadores | TD2 |
| Ventas por vendedor | Columnas | TD3 |
| Comparativa Online vs Tienda por categoría | Barras apiladas | TD4 |
| Ventas por trimestre | Columnas con % del total | TD5 |

**Requisitos de formato:**
- Todos los gráficos sin leyendas innecesarias
- Colores consistentes con la paleta del dashboard
- Título descriptivo en cada gráfico
- Sin líneas de cuadrícula en los gráficos

**Copilot en esta fase:**
```
"Crea un gráfico de barras horizontales con el total de ventas por Categoría,
ordenado de mayor a menor"
```
```
"¿Qué tipo de gráfico me recomiendas para comparar la evolución mensual de Online vs Tienda?"
```

---

### Fase 4 — KPIs (sesión 4.5)

**Objetivo:** calcular y presentar las métricas clave en tarjetas visuales.

KPIs obligatorios (calcula con fórmulas en la hoja `Cálculos`):

| KPI | Fórmula |
|-----|---------|
| Total ventas 2024 | SUMA(Total_Venta) |
| Número de operaciones | CONTARA(ID_Venta) |
| Ticket medio | Total_Venta / Operaciones |
| Mejor vendedor | ÍNDICE/COINCIDIR del máximo |
| Categoría top | ÍNDICE/COINCIDIR del máximo |
| Ventas Online | SUMAR.SI(Canal,"Online",Total_Venta) |
| Ventas Tienda | SUMAR.SI(Canal,"Tienda",Total_Venta) |

**Copilot en esta fase:**
```
"Calcula el ticket medio por canal (Online vs Tienda)"
```
```
"¿Cuál es el vendedor con mayor total de ventas y cuánto suma?"
```

---

### Fase 5 — Dashboard final (sesión 4.5)

**Objetivo:** ensamblar todos los elementos en una hoja limpia e interactiva.

#### Estructura del dashboard

```
┌─────────────────────────────────────────────────────────────────┐
│  DASHBOARD DE VENTAS 2024              │  [Seg. Canal] [Seg.Reg]│
├──────────┬──────────┬──────────┬───────┴──────────────────────────┤
│ Tot.Ventas│ Operac. │ Ticket M.│ Mejor Vendedor │ Categoría Top  │
├──────────┴──────────┴──────────┴─────────────────────────────────┤
│  Evolución mensual de ventas (gráfico de líneas — ancho completo) │
├─────────────────────────┬───────────────────────────────────────┤
│  Ventas por Categoría   │  Ventas por Vendedor                  │
│  (barras horizontales)  │  (columnas)                           │
├─────────────────────────┴───────────────────────────────────────┤
│  Online vs Tienda por Categoría (barras apiladas)               │
└─────────────────────────────────────────────────────────────────┘
```

#### Requisitos de diseño

- Paleta de colores: máximo 3 colores + blanco
- Sin cuadrícula visible (Vista → Líneas de cuadrícula: desactivar)
- Sin encabezados de fila/columna (Vista → Encabezados: desactivar)
- Título claro y visible en la parte superior
- Fuente consistente en todo el dashboard

#### Interactividad (mínimo obligatorio)

- 2 segmentadores conectados a todas las tablas dinámicas:
  - Canal (Online / Tienda)
  - Región (Norte / Sur / Este / Oeste / Centro)
- 1 escala de tiempo conectada a todas las tablas dinámicas
- Los KPIs deben actualizarse al filtrar (deben basarse en fórmulas que lean de las tablas dinámicas o en fórmulas SUMAR.SI que respeten los filtros de segmentador)

> **Nota técnica:** para que los KPIs respondan a los segmentadores, usa **OBTENER.DATOS.DINÁMICOS** o crea fórmulas que lean directamente de las celdas de las tablas dinámicas, no de los datos originales.

## Criterios de evaluación

| Criterio | Peso | Descripción |
|----------|------|-------------|
| Limpieza de datos | 15% | Datos correctos, sin duplicados, sin errores de formato |
| Tablas dinámicas | 20% | Correctas, con campos calculados y agrupación de fechas |
| Gráficos | 20% | Tipo adecuado, formato consistente, títulos descriptivos |
| KPIs | 20% | Fórmulas correctas, presentación visual clara |
| Diseño del dashboard | 15% | Layout, paleta, tipografía, coherencia visual |
| Interactividad | 10% | Segmentadores y timeline funcionando y conectados |

## Entregable

**Nombre del archivo:** `[Tu_Nombre]_Dashboard_Ventas_2024.xlsx`

- Hoja `Datos`: datos originales limpios

- Una nota con qué prompts de Copilot usaste y para qué
