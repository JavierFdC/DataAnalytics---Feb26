## 1. ¿Qué es una tabla dinámica?

Una tabla dinámica es una herramienta de resumen interactivo que permite agrupar, filtrar y agregar grandes volúmenes de datos sin modificar el dataset original. Con unos pocos clics puedes pasar de un listado de miles de filas a una vista resumida que responde una pregunta concreta.

**Ejemplos de preguntas que resuelve:**
- ¿Cuánto vendió cada vendedor en cada región?
- ¿Qué categoría genera más ingresos por trimestre?
- ¿Cuál es el ticket medio por canal de venta?
- ¿En qué meses el descuento promedio supera el 10%?

**¿Cuándo usar una tabla dinámica y cuándo no?**

| Situación | ¿Tabla dinámica? |
|-----------|-----------------|
| Resumir y agrupar datos por una o varias categorías | Sí |
| Comparar métricas entre grupos (vendedores, regiones, categorías) | Sí |
| Analizar evolución temporal (meses, trimestres, años) | Sí |
| Calcular una fórmula compleja que cruza múltiples tablas | No — mejor BUSCARV o Power Query |
| Mostrar el detalle de cada fila individual | No — mejor filtro directo sobre la tabla |

## 2. Preparar los datos

Una tabla dinámica es tan buena como los datos de origen. Antes de crearla:

- Los datos deben estar en **formato de tabla** (Ctrl+T) o en un rango continuo sin filas vacías
- Cada columna debe tener un **encabezado único y descriptivo** — los nombres de columna se convierten en los campos que arrastrarás
- No debe haber filas o columnas completamente vacías dentro del rango
- Los tipos de dato deben ser correctos: fechas como fechas, números como números (no texto)
- Los valores de texto deben ser consistentes: "Online" y "online" se tratarán como dos categorías distintas

> Si el dataset tiene problemas de calidad, aplica primero las técnicas de la sesión 4.1 antes de continuar.

## 3. Crear la tabla dinámica

Puedes crear una tabla dinámica de dos formas. El resultado es el mismo; elige según tu preferencia o la complejidad del análisis que necesitas.

### Opción A — Manualmente

1. Haz clic en cualquier celda dentro de la tabla de datos
2. Ve a **Insertar → Tabla dinámica**
3. Comprueba que el rango detectado es correcto (debe abarcar todos los datos)
4. Elige dónde colocarla: **nueva hoja** (recomendado para mantener el orden) o hoja existente
5. Haz clic en **Aceptar**

Se abre el **Panel de campos de tabla dinámica** a la derecha, con la lista de todas las columnas del dataset. A partir de aquí construyes el análisis arrastrando campos a las cuatro áreas (ver sección 4).

**Consejo:** si el panel de campos desaparece, haz clic dentro de la tabla dinámica para que vuelva a aparecer.

### Opción B — Con Copilot

Describe el análisis en lenguaje natural y Copilot genera la tabla dinámica directamente en una nueva hoja:

```
"Crea una tabla dinámica que muestre el total de ventas (Total_Venta) por Categoría y Canal"
```
```
"Crea una tabla dinámica con el total de ventas por Vendedor ordenada de mayor a menor"
```
```
"Genera una tabla dinámica con las ventas mensuales agrupadas por trimestre"
```

**Cuándo preferir Copilot para crear la tabla:**
- El análisis es sencillo y puedes describirlo en una frase
- Quieres explorar rápidamente varias vistas sin configurar cada una a mano
- Es tu primer contacto con tablas dinámicas y quieres ver un ejemplo antes de replicarlo

**Cuándo preferir hacerlo manualmente:**
- Necesitas un control preciso sobre el orden de los campos o los cálculos
- El análisis requiere varios campos calculados o configuraciones específicas
- Quieres asegurarte de entender cada campo que estás usando

> **Importante:** cuando Copilot crea la tabla dinámica, revisa siempre qué campos ha colocado en cada área antes de usar los resultados.

## 4. Configurar y enriquecer la tabla dinámica

Una vez creada la tabla — por cualquiera de los dos métodos — puedes configurarla y ampliarla. Todas las opciones de esta sección son independientes entre sí; aplica las que necesite tu análisis.

### 4.1 Las cuatro áreas en detalle

El panel de campos tiene cuatro zonas. Puedes arrastrar el mismo campo a varias áreas a la vez, y puedes poner varios campos en la misma área.

| Área | Función | Cuándo usarla | Ejemplo con nuestro dataset |
|------|---------|---------------|-----------------------------|
| **Filas** | Muestra cada valor único como una fila | Siempre — es la dimensión principal de análisis | Vendedor, Categoría, Región |
| **Columnas** | Añade una segunda dimensión en horizontal | Cuando quieres comparar grupos en paralelo | Canal (Online / Tienda) |
| **Valores** | Calcula la métrica para cada combinación | Siempre — es lo que se mide | Total_Venta (Suma), Unidades |
| **Filtros** | Añade un filtro global sobre toda la tabla | Cuando quieres ver el análisis para un subconjunto | Categoría, Año |

**Varios campos en el área Filas: jerarquías**

Puedes apilar varios campos en Filas para crear una jerarquía. El primer campo es el nivel más alto; los siguientes se despliegan dentro de él.

Ejemplo — arrastrar `Categoría` y luego `Producto` a Filas produce:
```
Electrónica
  ├─ Laptop
  ├─ Smartphone
  └─ Tablet
Ropa
  ├─ Camiseta
  └─ Pantalón
...
```
Usa el símbolo `+` / `-` junto a cada categoría para expandir o contraer el nivel inferior. Esto permite ver el resumen agregado y, cuando interesa, bajar al detalle sin crear una tabla nueva.

**Varios campos en el área Valores: múltiples métricas en paralelo**

Puedes añadir varios campos a Valores para mostrar más de una métrica por fila. Ejemplo: `Total_Venta` (Suma) y `Unidades` (Suma) en la misma tabla.

También puedes añadir el mismo campo dos veces y configurar cada instancia de forma distinta: una como Suma y otra como % del total general. Esto permite ver el valor absoluto y el peso relativo en la misma vista.

**El área Columnas: úsala con moderación**

Cada valor único del campo que pongas en Columnas genera una columna nueva. Si el campo tiene muchos valores distintos (por ejemplo, `Producto` con 12 productos), la tabla se vuelve muy ancha y difícil de leer. Reserva el área Columnas para campos con pocos valores: Canal (2), Trimestre (4), Región (5).

### 4.2 Funciones de agregación y "Mostrar valores como"

**Cambiar la función de agregación**

Por defecto, Excel usa SUMA para columnas numéricas y CONTAR para texto. Para cambiar:
1. Clic derecho sobre cualquier valor de la tabla → **Configuración de campo de valor**
2. En la pestaña **Resumir valores por**, selecciona la función:

| Función | Cuándo usarla | Ejemplo |
|---------|---------------|---------|
| Suma | Totales acumulados | Total de ventas por región |
| Promedio | Medias por grupo | Descuento medio por categoría |
| Recuento | Número de transacciones | Cuántas ventas hizo cada vendedor |
| Recuento de números | Solo cuenta celdas numéricas, ignora texto/vacíos | Útil para detectar huecos en datos numéricos |
| Máximo / Mínimo | Valor extremo del grupo | Mayor venta individual por vendedor |
| Producto | Multiplicación acumulada | Poco habitual en análisis de negocio |
| Desviación estándar | Dispersión dentro del grupo | Variabilidad del ticket por categoría |

**"Mostrar valores como": cálculos relativos sin fórmulas**

Esta opción (pestaña **Mostrar valores como** dentro de Configuración de campo de valor) transforma el valor calculado en una representación relativa. Es una de las funcionalidades más potentes y menos conocidas de las tablas dinámicas.

| Opción | Qué muestra | Ejemplo útil |
|--------|-------------|--------------|
| % del total general | Qué peso tiene cada celda sobre el total | Contribución de cada categoría al total de ventas |
| % del total de la fila | Qué porcentaje representa respecto a su fila | De cada vendedor, cuánto % viene de cada región |
| % del total de la columna | Qué porcentaje representa respecto a su columna | De cada región, cuánto % aporta cada vendedor |
| % del elemento principal | Porcentaje respecto al padre en una jerarquía | % de cada producto respecto a su categoría |
| Diferencia respecto a | Diferencia absoluta frente a un elemento de referencia | Variación de ventas respecto al mes anterior |
| % de diferencia respecto a | Variación porcentual frente a un elemento de referencia | Crecimiento mensual en % |
| Total acumulado en | Suma acumulada a lo largo de un campo | Ventas acumuladas mes a mes durante el año |

**Ejemplo práctico con nuestro dataset:**
Añade `Total_Venta` dos veces al área Valores:
- Primera instancia: Suma → muestra el importe absoluto
- Segunda instancia: % del total general → muestra el peso de cada fila sobre el total

Obtienes en una sola tabla tanto el importe como la participación de cada categoría, sin escribir ninguna fórmula.

### 4.3 Agrupar datos

**Agrupar por fechas**

Cuando arrastras una columna de fecha al área Filas o Columnas, Excel puede agrupar los datos por períodos en lugar de mostrar cada fecha individual.

*Agrupación automática:* en versiones recientes de Excel, al arrastrar `Fecha` a Filas aparece automáticamente agrupada por Años → Trimestres → Meses. Usa `+` / `-` para expandir o contraer cada nivel.

*Agrupación manual:*
1. Arrastra `Fecha` al área Filas
2. Clic derecho sobre cualquier fecha dentro de la tabla → **Agrupar**
3. Selecciona los niveles: Meses, Trimestres, Años (puedes marcar varios a la vez)
4. Haz clic en Aceptar

Cuando seleccionas varios niveles (por ejemplo Trimestres y Meses), Excel crea una jerarquía automática: puedes ver el resumen por trimestre y expandir para ver el detalle mensual.

> **Problema frecuente:** si la opción Agrupar aparece en gris, las fechas están almacenadas como texto en el dataset. Corrige el tipo de dato (sesión 4.1) y actualiza la tabla dinámica.

**Agrupar valores numéricos en rangos**

También puedes agrupar campos numéricos en intervalos. Útil para crear rangos de precio, rangos de descuento, etc.

Ejemplo — agrupar `Descuento_%` en rangos de 5 puntos:
1. Arrastra `Descuento_%` al área Filas
2. Clic derecho sobre cualquier valor → **Agrupar**
3. Define el valor mínimo, el máximo y el intervalo (ej: de 0 a 20, cada 5)
4. Haz clic en Aceptar

Obtienes filas como `0-5`, `5-10`, `10-15`, `15-20`, que permiten analizar el comportamiento de ventas por tramo de descuento.

### 4.4 Ordenar y filtrar

**Ordenar por etiqueta (A-Z) o por valor (mayor a menor)**

- Clic en la flecha del encabezado de fila → **Más opciones de ordenación**
- Para ordenar por valor: clic derecho sobre un número en la tabla → **Ordenar → De mayor a menor**

La ordenación por valor siempre se refiere al campo de Valores activo. Si tienes varios campos en Valores, comprueba sobre cuál haces clic derecho antes de ordenar.

**Filtrar elementos concretos**

- Clic en la flecha de **Etiquetas de fila** → desmarca los elementos que no quieres ver
- Los elementos excluidos siguen en el dataset original; solo se ocultan en la vista

**Filtros de valor (Top N, mayor que, entre…)**

Clic en la flecha de Etiquetas de fila → **Filtros de valor**:

| Filtro | Qué hace |
|--------|----------|
| Los 10 primeros | Muestra los N elementos con mayor o menor valor |
| Mayor que / Menor que | Muestra solo los elementos que superan o no llegan a un umbral |
| Entre | Muestra los elementos cuyo valor está dentro de un rango |
| Igual a / No igual a | Filtra por un valor exacto |

Ejemplo: mostrar solo los vendedores cuyo Total_Venta supere los 5.000 €:
→ Etiquetas de fila → Filtros de valor → Mayor que → 5000

**Filtro de página (área Filtros)**

Arrastra un campo al área Filtros para añadir un desplegable encima de la tabla. Permite cambiar el subconjunto analizado con un clic sin alterar la estructura.

Puedes seleccionar varios elementos a la vez activando la opción **Seleccionar varios elementos** dentro del desplegable.

> **Diferencia clave entre Filtros de etiqueta y Filtro de página:** el Filtro de etiqueta actúa sobre los valores que aparecen como filas o columnas; el Filtro de página actúa sobre todo el dataset antes de que la tabla haga cualquier cálculo.

---

### 4.5 Segmentadores y escala de tiempo

Los segmentadores y la escala de tiempo son la forma visual de filtrar una tabla dinámica: en lugar de usar los desplegables del área Filtros, el usuario hace clic en botones o arrastra un control. Son imprescindibles en cualquier dashboard interactivo.

**Insertar un segmentador**

1. Haz clic dentro de la tabla dinámica
2. Ve a **Analizar tabla dinámica → Insertar segmentador**
3. Selecciona los campos por los que quieres filtrar (ej: Categoría, Región, Canal)
4. Haz clic en Aceptar

Aparecen paneles de botones con los valores únicos de cada campo. Clic en un botón filtra la tabla; **Ctrl + clic** permite selección múltiple; el botón de borrar (esquina superior derecha del segmentador) quita el filtro.

Para ajustar el aspecto: clic en el segmentador → pestaña **Segmentador** → cambia el estilo de color o el número de columnas de botones.

**Insertar una escala de tiempo**

La escala de tiempo es un segmentador especializado para columnas de fecha: permite filtrar por período arrastrando un control deslizante.

1. Haz clic dentro de la tabla dinámica
2. **Analizar tabla dinámica → Insertar escala de tiempo**
3. Selecciona la columna de fecha (`Fecha`)
4. Haz clic en Aceptar

Usa el selector de granularidad (Días / Meses / Trimestres / Años) en la esquina superior derecha del control para cambiar el nivel de detalle temporal.

> Para conectar segmentadores a varias tablas dinámicas a la vez (muy útil en dashboards), consulta la sesión 4.4, donde se trata en profundidad junto con los filtros automáticos y avanzados.

### 4.6 Diseño y presentación

**Tres tipos de diseño**

La pestaña **Diseño → Diseño de informe** ofrece tres presentaciones:

| Diseño | Aspecto | Mejor para |
|--------|---------|------------|
| Compacto | Todos los campos de Filas en una sola columna, con sangría | Vista rápida, ahorra espacio horizontal |
| Esquema | Cada campo de Filas en su propia columna, con subtotales encima | Leer jerarquías con claridad |
| Tabular | Cada campo en su columna, sin sangría, subtotales debajo | Exportar o copiar datos a otra herramienta |

El diseño **Tabular** es el más limpio para copiar datos de la tabla dinámica a otro sistema, porque cada fila tiene todos los valores completos sin celdas vacías ni sangría.

**Subtotales y totales generales**

- **Diseño → Subtotales**: muestra, oculta o reposiciona los subtotales de cada grupo en el área Filas
- **Diseño → Totales generales**: activa o desactiva la fila de total al final y la columna de total a la derecha

En dashboards ejecutivos suele convenir desactivar los subtotales intermedios y dejar solo el total general para no sobrecargar la vista.

**Repetir etiquetas de elemento**

En diseño Tabular o Esquema, las celdas del primer nivel de jerarquía aparecen vacías cuando el grupo tiene varias filas. Para rellenarlas:
**Diseño → Diseño de informe → Repetir todas las etiquetas de elemento**

Esto es imprescindible si vas a usar los datos de la tabla dinámica en una fórmula o en Power Query.

**Estilo visual**

Pestaña **Diseño** → galería de estilos. Los estilos "claro" son más limpios para presentaciones; los "medio" añaden color en los encabezados; los "oscuro" aportan más contraste pero son más agresivos visualmente.

**Formato numérico**

Clic derecho sobre cualquier valor → **Formato de número** → elige la categoría (moneda, porcentaje, número con separadores…). Este formato se aplica a todos los valores de ese campo en toda la tabla, no solo a la celda seleccionada.

### 4.7 Campos calculados

Un campo calculado es una nueva columna que se calcula dentro de la tabla dinámica combinando campos del dataset. No modifica los datos originales.

**Cómo crear un campo calculado:**
1. Haz clic dentro de la tabla dinámica
2. Ve a **Analizar tabla dinámica → Cálculos → Campos, elementos y conjuntos → Campo calculado**
3. Escribe un nombre descriptivo (ej: `Ticket_Medio`)
4. Escribe la fórmula usando los nombres exactos de las columnas del dataset:
   ```
   = Total_Venta / Unidades
   ```
5. Haz clic en **Agregar** → **Aceptar**

El campo aparece en el área Valores y se calcula para cada combinación de filas y columnas.

**Ejemplos de campos calculados útiles con nuestro dataset:**

| Nombre | Fórmula | Para qué sirve |
|--------|---------|----------------|
| Ticket_Medio | `= Total_Venta / Unidades` | Importe medio por unidad vendida |
| Precio_con_descuento | `= Precio_Unitario * (1 - Descuento_% / 100)` | Precio efectivo tras aplicar el descuento |
| Margen_estimado | `= Total_Venta * 0.3` | Estimación rápida de margen al 30% |

**Limitaciones importantes de los campos calculados:**

- La fórmula opera sobre los **totales agregados de cada grupo**, no sobre cada fila individual del dataset. Esto puede dar resultados distintos a calcular la misma fórmula en el dataset y luego agregar.

  Ejemplo: `= Total_Venta / Unidades` en un campo calculado divide la *suma de ventas del grupo* entre la *suma de unidades del grupo*, que es el ticket medio ponderado correcto. Pero si alguna fila tiene 0 unidades, puede producir errores en el total.

- No pueden referenciar celdas externas a la tabla dinámica ni usar funciones de Excel que dependan de rangos (BUSCARV, SUMAR.SI, etc.)
- No son compatibles con todas las opciones de "Mostrar valores como"
- Para eliminar un campo calculado: **Analizar → Campo calculado → selecciona el campo → Eliminar**

### 4.8 Explorar el detalle: drill-down

Cuando haces **doble clic sobre cualquier valor** de la tabla dinámica, Excel crea automáticamente una nueva hoja con todas las filas del dataset original que contribuyen a ese número.

Esto se llama *drill-down* y es muy útil para investigar un dato concreto: si el total de ventas de Electrónica en el Norte parece demasiado alto, haces doble clic y ves exactamente qué transacciones lo forman.

> Para desactivar esta función: clic derecho dentro de la tabla → **Opciones de tabla dinámica → pestaña Datos → desmarcar "Habilitar mostrar detalles"**.

---

### 4.9 Actualización y gestión del origen de datos

**Actualizar manualmente**

La tabla dinámica no se refresca sola cuando modificas el dataset. Para actualizarla:
- Clic derecho dentro de la tabla → **Actualizar**
- O ve a **Analizar tabla dinámica → Actualizar → Actualizar todo** (refresca todas las tablas del libro)

**Cambiar el rango de origen**

Si añades filas al dataset más allá del rango original:
**Analizar tabla dinámica → Cambiar origen de datos** → ajusta el rango para incluir las nuevas filas.

Este problema desaparece si el origen es una **tabla con formato** (Ctrl+T): las tablas de Excel se expanden automáticamente cuando añades filas, y la tabla dinámica las incluye al actualizar.

**Varias tablas dinámicas sobre el mismo origen**

Puedes crear tantas tablas dinámicas como necesites sobre el mismo dataset, cada una con una configuración diferente. Todas se actualizan a la vez con **Actualizar todo**.

## 5. Analizar resultados con Copilot

Una vez que tienes una tabla dinámica creada, Copilot puede ayudarte a interpretar los datos, detectar patrones o añadir métricas adicionales sin necesidad de configurar más campos manualmente.

### Interpretar y comparar
```
"¿Qué región tiene el mayor promedio de ventas por operación?"
```
```
"Compara las ventas del canal Online frente a Tienda por categoría y dime cuál tiene mejor rendimiento"
```
```
"¿Qué vendedor tiene el mayor número de ventas (no importe, sino número de operaciones)?"
```

### Detectar patrones y anomalías
```
"¿Hay alguna categoría cuyas ventas hayan caído entre el primer y el segundo trimestre?"
```
```
"¿Qué región muestra el mayor descuento promedio? ¿Podría estar afectando al margen?"
```

### Añadir métricas
```
"Añade una columna en la tabla dinámica que muestre el ticket medio (Total_Venta / Unidades)"
```
```
"Añade el porcentaje que representa cada categoría sobre el total de ventas"
```

> **Nota:** Copilot sugiere la acción pero no edita directamente las celdas. Revisa siempre el resultado antes de aceptar, especialmente en campos calculados donde la lógica de agregación puede no ser la esperada.

## Errores frecuentes

| Error | Causa probable | Solución |
|-------|---------------|----------|
| Los valores aparecen como "Recuento" en lugar de "Suma" | La columna numérica tiene alguna celda con texto | Corrige el tipo de dato en el dataset y actualiza |
| No aparece la opción de agrupar fechas | Las fechas están almacenadas como texto | Convierte la columna a tipo Fecha en el dataset original |
| La tabla dinámica no se actualiza con los nuevos datos | Hay que actualizar manualmente | Clic derecho → Actualizar, o amplía el rango de origen |
| Copilot no reconoce los nombres de columna | El encabezado tiene espacios o caracteres especiales | Renombra las columnas usando nombres simples sin espacios |
| El campo calculado da resultados inesperados | Se aplica sobre totales, no sobre filas individuales | Revisa la lógica de la fórmula teniendo en cuenta la agregación |

Crea una tabla dinámica que responda: **¿qué categoría vende más en importe y en unidades?**

- Añade el porcentaje sobre el total general para Total_Venta

Crea una tabla dinámica que responda: **¿cómo evolucionan las ventas a lo largo del año por canal?**
- Filas: `Fecha` agrupada por Trimestre y Mes
- Columnas: `Canal`
- Valores: `Total_Venta` (Suma)

Responde: ¿qué trimestre concentra más ventas? ¿El canal Online o Tienda es más estable a lo largo del año?

### Ejercicio 3 — Cruce de dimensiones (manual)
Crea una tabla dinámica que responda: **¿qué vendedor lidera en cada región para la categoría Electrónica?**
- Filas: `Vendedor`
- Columnas: `Región`
- Valores: `Total_Venta` (Suma)
- Filtro de página: `Categoría` → filtra por "Electrónica"

Prueba a cambiar el filtro de página a otra categoría y observa cómo cambia el ranking de vendedores.

### Ejercicio 4 — Campo calculado (manual)

### Ejercicio 6 — Segmentadores y escala de tiempo
