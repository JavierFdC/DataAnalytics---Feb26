## 1. Probabilidad — conceptos fundamentales

La **probabilidad** es una medida numérica de la incertidumbre. Asigna un valor entre 0 y 1 a la posibilidad de que ocurra un evento.

```
P(A) = Número de casos favorables / Número de casos posibles
```

| Valor | Interpretación |
|-------|---------------|
| P = 0 | El evento es imposible |
| P = 1 | El evento es seguro |
| 0 < P < 0,5 | Ocurre menos de la mitad de las veces |
| P = 0,5 | Igual de probable que ocurra o que no ocurra |
| 0,5 < P < 1 | Ocurre más de la mitad de las veces |

### Ejemplo en ventas_retail.csv

El dataset contiene 12.000 transacciones. De ellas, 1.800 incluyen un descuento superior al 20%.

```
P(descuento > 20%) = 1.800 / 12.000 = 0,15 = 15%
```

Esto no significa que la próxima transacción tenga exactamente un 15% de posibilidades. Significa que, **en condiciones similares y a largo plazo**, 1 de cada 7 transacciones llevará un descuento superior al 20%.

## 2. Probabilidad compuesta

Cuando trabajamos con más de un evento, necesitamos combinar probabilidades.

### Regla de la adición (evento A o evento B)

```
P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
```

El término P(A ∩ B) evita contar dos veces los casos en que ocurren ambos.

**Caso especial — eventos mutuamente excluyentes** (no pueden ocurrir a la vez):
```
P(A ∪ B) = P(A) + P(B)
```

### Regla de la multiplicación (evento A y evento B)

```
P(A ∩ B) = P(A) × P(B|A)
```

Donde P(B|A) es la probabilidad de B dado que ya ocurrió A.

**Caso especial — eventos independientes** (uno no afecta al otro):
```
P(A ∩ B) = P(A) × P(B)
```

### Ejemplo en ventas

Del dataset de 12.000 transacciones:
- P(devolución) = 0,08 → 960 transacciones resultaron en devolución
- P(descuento > 20%) = 0,15 → 1.800 transacciones con descuento alto
- P(devolución ∩ descuento > 20%) = 0,03 → 360 transacciones con ambos

**¿Cuál es la probabilidad de devolución O descuento > 20%?**
```
P(devolución ∪ descuento > 20%) = 0,08 + 0,15 − 0,03 = 0,20 = 20%
```

## 3. Probabilidad condicional

La **probabilidad condicional** P(B|A) es la probabilidad de que ocurra B sabiendo que ya ha ocurrido A. Es uno de los conceptos más importantes en análisis de datos.

```
P(B|A) = P(A ∩ B) / P(A)
```

### Ejemplo en ventas

Del dataset de 12.000 transacciones:
- 960 resultaron en devolución
- 360 tuvieron devolución Y descuento > 20%

**¿Cuál es la probabilidad de devolución dado que hubo descuento > 20%?**

```
P(devolución | descuento > 20%) = 360 / 1.800 = 0,20 = 20%
```

Comparado con el 8% de devoluciones en el total, aplicar descuentos altos duplica la tasa de devolución. Este dato podría llevar a revisar la política de descuentos: ¿se están usando para colocar productos con mayor probabilidad de devolución?

### Por qué importa la probabilidad condicional en análisis

En datos, casi nunca nos interesa la probabilidad global. Nos interesa la probabilidad **dentro de un segmento**:

| Pregunta global (menos útil) | Pregunta condicional (más útil) |
|-----------------------------|--------------------------------|
| ¿Qué % de clientes se dan de baja? | ¿Qué % de clientes con menos de 3 meses se dan de baja? |
| ¿Qué % de productos se devuelven? | ¿Qué % de productos de la categoría electrónica se devuelven? |
| ¿Qué % de solicitudes de crédito se aprueban? | ¿Qué % de solicitudes de autónomos se aprueban? |

## 4. Teoría de conjuntos

La teoría de conjuntos proporciona el lenguaje para combinar y comparar grupos de datos. Sus operaciones son la base de los filtros, las uniones y las intersecciones que se usan en SQL y en cualquier herramienta de análisis.

### Operaciones principales

| Operación | Símbolo | Definición | Equivalente en datos |
|-----------|---------|-----------|---------------------|
| **Unión** | A ∪ B | Elementos que están en A, en B o en ambos | OR en filtros; UNION en SQL |
| **Intersección** | A ∩ B | Elementos que están en A y también en B | AND en filtros; INNER JOIN en SQL |
| **Complementario** | Aᶜ | Elementos que NO están en A | NOT en filtros |
| **Diferencia** | A − B | Elementos que están en A pero no en B | LEFT JOIN con WHERE B IS NULL |

### Ejemplo en ventas_retail.csv

Sea:
- **A** = transacciones del canal online
- **B** = transacciones con descuento > 20%
- **C** = transacciones de la categoría Electrónica

| Expresión | Qué representa | Pregunta de negocio |
|-----------|---------------|---------------------|
| A ∩ B | Ventas online con descuento alto | ¿El canal online aplica más descuentos que otros canales? |
| B ∩ C | Ventas de electrónica con descuento alto | ¿Los descuentos se concentran en la categoría electrónica? |
| A ∪ C | Ventas online o de electrónica | ¿Cuántas transacciones pertenecen a alguno de estos dos grupos? |
| C − B | Ventas de electrónica sin descuento alto | ¿Cuánta electrónica se vende a precio completo? |
| Aᶜ | Ventas que no son del canal online | ¿Qué representa el canal físico y telefónico? |

### Diagramas de Venn

Los diagramas de Venn representan visualmente las operaciones de conjuntos. Son útiles para comunicar la segmentación de una población a audiencias no técnicas.

## 5. Combinatoria

La combinatoria cuenta el número de formas posibles de organizar o seleccionar elementos. En análisis de datos es útil para entender el espacio de posibilidades: cuántas combinaciones de segmentos existen, cuántas rutas posibles hay, cuántos escenarios debe cubrir un test.

### Principio de multiplicación

Si una decisión tiene *m* opciones y una segunda tiene *n* opciones, el total de combinaciones posibles es *m × n*.

**Ejemplo:** un dashboard de ventas tiene 4 regiones, 6 categorías de producto y 3 canales. El número de combinaciones de filtro distintas es:
```
4 × 6 × 3 = 72 combinaciones posibles
```

### Permutaciones — el orden importa

Una **permutación** es una ordenación de elementos en la que el orden es relevante.

```
P(n, r) = n! / (n − r)!
```

**Ejemplo:** ¿de cuántas formas distintas se puede asignar el primer, segundo y tercer puesto del ranking de vendedores del trimestre entre 8 candidatos?
```
P(8, 3) = 8! / (8−3)! = 8! / 5! = 8 × 7 × 6 = 336 formas distintas
```

### Combinaciones — el orden no importa

Una **combinación** es una selección de elementos en la que el orden no es relevante.

```
C(n, r) = n! / (r! × (n − r)!)
```

**Ejemplo:** ¿de cuántas formas distintas se pueden elegir 3 productos de un catálogo de 8 para incluirlos en una campaña de promoción?
```
C(8, 3) = 8! / (3! × 5!) = 56 combinaciones posibles
```

### Permutaciones vs. combinaciones

| | Permutación | Combinación |
|--|-------------|-------------|
| ¿El orden importa? | Sí | No |
| Ejemplo | Podio de ventas (1º, 2º, 3º son distintos) | Equipo de trabajo (no importa quién se eligió primero) |
| Fórmula | n! / (n−r)! | n! / (r! × (n−r)!) |
| Resultado | Mayor o igual que combinación | Menor o igual que permutación |

## 6. Probabilidad en el análisis de datos — aplicaciones reales

| Aplicación | Concepto usado | Ejemplo |
|-----------|---------------|---------|
| **Segmentación de riesgo** | Probabilidad condicional | Clientes con mayor probabilidad de impago según su perfil |
| **Detección de anomalías** | Probabilidad de cola | Una transacción en el 0,1% superior de importe requiere revisión |
| **Pruebas A/B** | Probabilidad e independencia | ¿El nuevo diseño del botón mejora la conversión o es variación aleatoria? |
| **Cobertura de pruebas** | Combinatoria | ¿Cuántas combinaciones de región + canal + categoría debo verificar? |
| **Filtros en SQL / BI** | Teoría de conjuntos | WHERE canal = 'online' AND descuento > 20 |
